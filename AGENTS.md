# AGENTS.md

## Context
このディレクトリは relative-ear-app の開発ルートである。

## Required Reading
作業開始前に `./SPEC.md` を読むこと。
実装、設計変更、レビュー、タスク分解の際は `./SPEC.md` を仕様の正本として扱うこと。

## Rules
- 仕様に曖昧さがある場合は `SPEC.md` を優先する
- `SPEC.md` に未記載の前提を置く場合は明示する
- 仕様と実装に差分がある場合は、その差分を指摘してから進める

## Post-Task
- 仕様変更を伴う作業を完了した場合は、`SPEC.md` の該当箇所を更新すること
- 更新が不要と判断した場合も、その理由を明示すること
